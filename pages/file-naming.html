File & Folder Naming Conventions

Tips for keeping your site clean, organized, and easy to manage.

Use lowercase file names with no spaces

Avoid spaces and uppercase letters to prevent errors and keep things consistent. Use dashes or underscores if needed.

Do: index.html or contact-form.css
Don’t: My Page.html or StyleSheet.CSS

Always name your homepage index.html

This is the default page browsers will look for when loading a folder or domain. It should be your landing or home page.

Organize images in a dedicated folder

Keep your visuals tidy by putting them in a folder like /img/ or /images/.

Example: /img/gallery1.jpg, /images/headshot.png

Be consistent and descriptive

Use logical names like:

  • cv.pdf
  • bio.txt
  • gallery-portrait-01.jpg
  • project-resume.pdf

Avoid ambiguous names like doc1.pdf or image123.jpg.

Keep file extensions standard

Use .html for web pages
Use .css for stylesheets
Use .js for JavaScript
Use .jpg, .png, or .svg for images

Avoid duplicate file names

No two files in the same folder should have the same name — it will overwrite one or confuse your code.

Back up your files regularly

Use GitHub, Google Drive, or a USB flash drive to keep copies of your work in case something breaks.
